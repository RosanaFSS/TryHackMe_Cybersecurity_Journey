<h2>Task 6 . Password Protection<a id='6'></a></h2>
<p>Many security guidelines advise against using identical passwords across different systems to maintain complexity and discourage simple passwords like <strong>Password123</strong>. Microsoft Entra <strong>Password Protection</strong> can identify and prevent the use of commonly weak passwords and their variations while also allowing the blocking of other weak terms that are particular to your organization.</p>

<h6 align="center"><img width="800px" src="https://github.com/user-attachments/assets/8ef0bd0a-3ce1-46dd-a246-84d657a23a30"><br>This image and all the theoretical content of the present article is TryHackMeÂ´s property.</h6>

<h3>Global Banned Password List</h3>
<p>This is a collection of commonly used or weak passwords that Microsoft automatically blocks to improve security. The list is created by examining Microsoft Entra security telemetry data to find frequently used or compromised passwords. The aim is to stop users from using passwords that are widely used or known to be susceptible to password attacks.<br>

- When weak passwords are identified, they are included in the <strong>lobal banned password list</strong>.<br>
- The global banned password list is <strong>automatically enforced</strong> for all Microsoft Entra users.<br>
- The global banned password list is not visible to customers, so it cannot be disabled or modified; it is always enabled by default.<br>
- When users <strong>reset</strong> or <strong>change</strong> their password in a Microsoft Entra tenant, the new password is <strong>validated</strong> against the global banned password list to ensure stronger passwords are used across all Microsoft Entra tenants.</p>

<h3>Custom Banned Password List</h3>
<p>Organizations have the option to develop a custom banned password list specifically designed to meet their unique security needs if they wish to enforce an extra layer of security. Furthermore, password protection can be enabled for on-premises environments by integrating it with Windows Server Active Directory. With this in place, password changes or resets are then validated against the combined set of these banned password lists.<br>

- A list of words, one per line, to prevent users from using their passwords.<br>
- Words specific to the organization, including products, trademarks, industries, local cities and towns, and local sports teams.<br>
- The list can contain up to 1000 words. It is case-insensitive, and common character substitutions (e.g., o for 0) are automatically considered.<br>

<strong>Example</strong>: Consider an organization named "Weyland". There is <strong>no need</strong> to try to block <strong>specific variations</strong> of the terms below:<br>

- Weyland!1<br>
- Weyland@Nostromo<br>
- NostromoSulaco<br>
- !LV426<br>

Instead, only use <strong>key base terms</strong> to be blocked. The <strong>password validation algorithm</strong> then automatically blocks weak variants and combinations.</p>

<h6 align="center"><img width="800px" src="https://github.com/user-attachments/assets/4e577d07-23b1-4863-883b-206d866348d1"><br>This image and all the theoretical content of the present article is TryHackMeÂ´s property.</h6>

<h3>Password Spray Attacks</h3>
<p>A <strong>password spraying attack</strong>strong> is a type of <strong>brute force attack</strong>strong> in which an attacker tries the same password across multiple accounts before switching to a different password and repeating the method. This approach can be successful because many individuals opt for simple, easy-to-guess passwords like "password123".<br>

Most password spray attacks typically do not try to compromise a single account more than a few times because this tends to raise the chance of being detected, either through account lockouts or other methods. Instead, most password spray attacks use only a limited set of commonly weak passwords against each account within an organization. This strategy allows the attacker to swiftly identify an account that can be easily compromised while evading potential detection limits.<br>

Microsoft Entra <strong>Password Protection</strong> can block all known weak passwords that are likely to be used in password spray attacks since it's based on real-world security telemetry data from Microsoft Entra ID used to build the global banned password list.</p>

<h2>Task 7 . Self-Service Password Reset (SSPR)<a id='7'></a></h2>
<p>Self-service password reset (SSPR) is a feature that allows users to change or reset their passwords without needing help from an administrator or the help desk. If a userâ€™s account is locked or they cannot remember their password, they can simply follow the instructions to regain access and reset their passwords.</p>

<h3>How Does SSPR Work?</h3>
<p>Users can initiate the reset or change their password using the SSPR portal. Then, the user is prompted to enter a user ID and pass a captcha. Microsoft Entra ID now verifies that the user can use SSPR by doing the following checks:<br>

- If SSPR is enabled.
- If the right authentication methods are defined on the user's account per administrator policy
- If the user's password is managed on-premises<br>- It checks if the Microsoft Entra tenant is using federated, pass-through authentication, or password hash synchronization.<br>

If all is well, the user is guided through resetting or changing their password.</p>

<h3>Authentication Methods for SSPR</h3>
<p>When users are enabled for SSPR, they must register using at least one authentication method. However, two or more authentication methods are highly recommended to give users more flexibility while resetting their passwords.<br>

The following authentication methods are available for SSPR:<br>

- Mobile app notification<br>
- Mobile app code<br>
- Office phone<br>
- Email<br>
- Security questions<br>
- Mobile phone (SMS only)<br>

Users can only reset their password if they have registered an authentication method that the administrator has enabled.</p>

<h3>How to Enable SSPR?</h3>
<p>From an administrator perspective, at least an Authentication Policy Administrator role is required to enable self-service password reset (SSPR).<br>

- Log in to Microsoft Entra admin center<br>
- Navigate to <code>Protection</code> >  <code>Password reset</code></p>
